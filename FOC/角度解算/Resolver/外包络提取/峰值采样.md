峰值采样法（Peak Sampling）是旋变软解码中最直观、计算量最小的一种包络提取方法。它的核心思想是：利用ADC在载波信号的**峰值时刻**（即正峰值或负峰值）进行采样，此时调幅信号的幅度就等于包络值，从而省去了复杂的乘法或积分运算。

下面从原理、实现方式、关键难点、优缺点以及工程实践等方面展开详细说明。

实际工程中，通常采用以下两种方式之一：

- **单次采样**：仅采样正峰值（或负峰值），获得包络值。这种方法简单，但易受共模噪声和直流漂移影响。

## 2. 数字实现架构

### 2.1 采样时刻的生成

由于载波由MCU产生，可以利用产生励磁PWM的定时器来同步ADC触发。典型流程如下：

1. **MCU输出励磁信号**：通过定时器输出一个高频PWM（例如10kHz），经滤波后得到正弦波激励旋变。通常使用定时器的比较匹配事件来翻转PWM，同时该定时器也作为ADC触发的基准。
2. **触发ADC**：设置定时器的某个比较值（例如周期中点）作为ADC触发源。为了对准峰值，需要考虑硬件链路的延迟（励磁滤波、旋变线圈、调理电路等）。
3. **延迟补偿**：在定时器触发事件后插入一个可调的延时（用另一个定时器或软件延时），使得ADC采样时刻正好落在实际信号峰值处。该延时通常通过实验标定。
4. **采样两路信号**：在触发时刻，ADC同时（或连续快速）采样正弦绕组和余弦绕组的电压。

### 2.2 正负峰值交替采样

若采用差分方式，通常在每个载波周期内交替采样正峰值和负峰值：

- 在载波正半周期，采样一次正峰值，得到 ![](https://cdn.nlark.com/yuque/__latex/ea74a92def5eac956a71478f47c078dd.svg)。
- 在载波负半周期，采样一次负峰值，得到 ![](https://cdn.nlark.com/yuque/__latex/bd1016bacf8eb02e363b516e3eaed239.svg)。
- 然后计算差值：![](https://cdn.nlark.com/yuque/__latex/17c73561a463f2388d15401831af524f.svg)，同理得 ![](https://cdn.nlark.com/yuque/__latex/7cd43d57107c72c5c7ad362585b7da4e.svg)。

这种交替采样要求两个半周期的采样时刻均精确对准各自的峰值，且两路ADC通道的一致性要好。

### 2.3 数据处理

- 将采得的原始值减去事先标定的偏移（零位偏移），得到校正后的包络。
- 若采用差分法，则无需单独减偏移，因为正负相减自动抵消了直流偏置。
- 归一化：将两路包络除以其幅值（例如通过最大值标定），得到 ![](https://cdn.nlark.com/yuque/__latex/605d0e1cc65b9501482b9a107da21e4c.svg) 和 ![](https://cdn.nlark.com/yuque/__latex/6f68004a3cffdcd2dfadef558dc37e93.svg)。
- 角度计算：用 `atan2` 或跟踪观测器求取角度和速度。

## 3. 关键难点与误差分析

峰值采样法对时序要求极为严格，误差主要来源于以下几个方面：

### 3.1 相位延迟的标定与补偿

- **励磁链路的延迟**：PWM输出经RC滤波、驱动、旋变线圈、运放调理后，实际信号峰值相对于PWM触发点存在滞后。如果ADC采样点未对齐峰值，采到的幅值会偏低（例如采样点偏离 ![](https://cdn.nlark.com/yuque/__latex/becc4fcd9d0fe1d4c9363c6e000c53ed.svg)，则采到的值为 ![](https://cdn.nlark.com/yuque/__latex/83a44ece984b0916e3d8bafa73aa8872.svg)，带来幅值误差）。
- **温度漂移**：硬件延迟会随温度变化，需做温度补偿或采用自适应方法。

### 3.2 ADC采样与保持时间

ADC采样需要一定的时间（采样保持电容充电）。如果信号变化太快，可能无法建立到真实值，尤其在峰值附近斜率变化剧烈时。应确保ADC采样时间足够长，并选用高输入阻抗的调理电路。

### 3.3 噪声干扰

- 峰值采样只在单个时间点取值，如果该时刻存在尖峰噪声（如开关电源干扰），将直接导致错误的包络值。因此，系统需要良好的硬件滤波，或在软件中加入简单的滤波（如中值滤波、滑动平均）。

### 3.4 两路采样的同步性

对于正弦和余弦绕组，ADC应在同一时刻采样，以保证两者相位一致。若使用多通道ADC顺序采样，需考虑通道间的采样延时，必要时进行插值补偿。

### 3.5 低速下的幅值衰减

当转子角度使得包络接近零时（例如 ![](https://cdn.nlark.com/yuque/__latex/a551916f208bbec2b8bf371ae4b9de0e.svg)），信号幅值很小，信噪比极低，角度计算会抖动。此时仅靠峰值采样难以得到稳定角度，通常需结合角度观测器（如PLL或观测器）来平滑输出。

## 4. 优缺点总结

|  |  |
| --- | --- |
| 优点 | 缺点 |
| **计算量极小**：只需几次ADC读取和简单算术，适合资源受限的MCU。 | **时序要求苛刻**：必须精确对准峰值，对硬件延迟敏感，调试难度大。 |
| **无滤波器延迟**：实时性最好，适合高速控制环路。 | **抗噪声能力弱**：单点采样易受瞬时干扰。 |
| **实现简单**：代码量少，易于理解和维护。 | **受温度影响**：相位漂移会导致幅值误差，需补偿。 |
| **可配合差分采样消除共模噪声**。 | **低速性能差**：小信号时信噪比低，角度抖动大。 |

---

## 5. 工程实践建议

- **相位校准**：在系统初始化和定期自检时，通过软件扫描不同的采样延时，找到使包络幅值最大的延时值，并记录下来。也可利用旋变的自标定功能（如锁定转子在已知角度）来校准。
- **硬件设计**：确保信号链的带宽足够，避免信号在峰值附近畸变；增加必要的RC滤波以抑制高频噪声，但需注意滤波带来的额外相位延迟。
- **软件滤波**：对采得的包络值可进行简单的滑动平均或一阶低通滤波，以抑制随机噪声，同时控制滤波延迟在可接受范围内。
- **结合其他方法**：在低速区，可切换到积分法或相干解调以提高信噪比；在高速区使用峰值采样保证快速响应，即混合解调策略。

---

## 6. 总结

峰值采样法利用载波峰值的特性，通过精确时序采样直接获取包络，是旋变软解码中最简单高效的方法之一。它特别适用于对实时性要求极高、MCU资源有限且硬件调理良好的场合。然而，其成功实现的关键在于对**采样时刻的精确控制**以及对**相位延迟的准确补偿**。在实际应用中，需要结合硬件设计、软件校准和必要的滤波手段，才能发挥其优势，获得稳定可靠的角度输出。
