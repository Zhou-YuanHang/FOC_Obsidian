# 1. 硬件前提

在进入软件算法之前，硬件上通常需要做预处理：

- **ADC采样同步**：这是软解码成败的关键。因为载波频率很高（如10kHz），如果ADC采样点落在载波的过零点或非峰值点，包络幅值会很低或失真。

- **策略**：通常利用MCU的高精度PWM或定时器，在载波信号的**正峰值**和**负峰值**处触发ADC采样。
- **差分采样**：为了提高信噪比，建议采样“峰值-谷值”做差分。在载波正峰值时刻采样得到 ![](https://cdn.nlark.com/yuque/__latex/22ffb766df821e2144a107192f2aa170.svg)，在180°后的负峰值采样得到![](https://cdn.nlark.com/yuque/__latex/cedf160e136fe0e62c6961f401e7b808.svg)，则实际包络幅值=![](https://cdn.nlark.com/yuque/__latex/e75f24a11b21c04f021d828b2db10f6f.svg) （或除以2）。

## 2.2. 峰值采样

最常用，适合资源紧张的 MCU

如果可以通过硬件 PWM 精准控制 ADC 的采样时刻，使其能够严格对准载波的波峰波谷值，那么就可以直接提取外包络

- **采样逻辑**：

- 在正峰值时刻触发 ADC，得到![](https://cdn.nlark.com/yuque/__latex/fbcd68e4d8e703ab2295e95a46b2bd8d.svg)
- （可选）在负峰值时刻触发 ADC，得到![](https://cdn.nlark.com/yuque/__latex/5cfa17c8c0ea8b5e1b38d996c3aa4319.svg)

- **合成**：

- 正峰值：外包络=![](https://cdn.nlark.com/yuque/__latex/f3f0db1fc6b739ca47b8207b8414f9e0.svg)（包含直流偏置需校准）
- 正负峰值：外包络 = ![](https://cdn.nlark.com/yuque/__latex/d59153e9c93660ac6276c1e98090cd73.svg)。这种差分方式可以消除共模噪声和温漂

- **优点：**计算量小，实时性高
- **难点：相位对齐**。励磁信号经过旋变线圈、运放调理电路后会有相位延迟。需要软件调节PWM触发点，确保ADC采样时刻是在信号调理后的实际峰值处，而非软件发出的载波理论峰值处。

## 2.3. 过采样与正交解调（矢量跟踪）

在一些高端软解码算法（如基于锁相环PLL的软解码）中，提取包络和角度计算是同步进行的。

1. 不直接提取包络，而是构建一个观测器。
2. **误差计算**：利用“实际采样值”与“基于当前估计角度重构出的预期采样值”做差。
3. **闭环调节**：通过PI调节器驱动观测器，使重构值逼真实值。
4. 这种方法的包络提取隐含在闭环的误差信号处理中，通常不需要显式的低通滤波器，但需要对电机模型和旋变模型有较深的理解。

# 3. 关键难点与注意事项

在软解码中，提取外包络时容易遇到以下几个问题：

- **幅值衰减与畸变：**旋变输出信号进入MCU前通常经过RC滤波或运放调理。如果载波频率较高，而运放的压摆率或ADC采样保持时间不足，会导致采样到的峰值比实际峰值低，表现为“包络变形”。需要确保信号链的带宽足够（通常建议信号链带宽是载波频率的10倍以上）。

- **直流偏置（Offset）校准：**由于ADC参考电压、运放零漂等因素，采样得到的 ![](https://cdn.nlark.com/yuque/__latex/605d0e1cc65b9501482b9a107da21e4c.svg) 和 ![](https://cdn.nlark.com/yuque/__latex/6f68004a3cffdcd2dfadef558dc37e93.svg) 可能带有直流分量。直流分量会导致角度计算出现周期性误差（特别是导致速度纹波）。提取包络后，必须进行软件校准：

![](https://cdn.nlark.com/yuque/__latex/63807429850216017883e7af9314ff27.svg)

通常在电机静止时（或通过软件自检）完成校准。

- **正交性误差：**理想情况下 ![](https://cdn.nlark.com/yuque/__latex/605d0e1cc65b9501482b9a107da21e4c.svg) 和 ![](https://cdn.nlark.com/yuque/__latex/6f68004a3cffdcd2dfadef558dc37e93.svg) 应该正交。但由于旋变安装偏差或硬件通道延迟不一致，两者可能存在相位差。软解码中可以在提取包络后，通过坐标旋转（CORDIC）算法中的补偿参数进行修正。
- **噪声抑制：**在低速或零速时，外包络信号幅值很小（![](https://cdn.nlark.com/yuque/__latex/605d0e1cc65b9501482b9a107da21e4c.svg) 接近0），此时信噪比极差。仅靠提取包络是不够的，后续的角度观测器（如跟踪观测器或滑模观测器）至关重要，否则角度抖动会非常大。

# 4. 流程总结

一个典型的软解码流程（峰值采样法）如下：

1. MCU生成高频PWM ![](https://cdn.nlark.com/yuque/__latex/d59084f8d5c6229967c0d93bd3204205.svg) 经过驱动电路 ![](https://cdn.nlark.com/yuque/__latex/d59084f8d5c6229967c0d93bd3204205.svg) 产生励磁载波 ![](https://cdn.nlark.com/yuque/__latex/9afc11e960115030fa57554cf1d4a6dc.svg)。
2. ADC触发：利用PWM的周期匹配或比较匹配事件，延时 ![](https://cdn.nlark.com/yuque/__latex/1cb19b2e1c3771fe60fcadf7bd1daf7d.svg)（补偿硬件延迟）后触发ADC。
3. 采样：同时采集旋变正弦绕组和余弦绕组的电压。
4. 差分合成：如果是正负峰值交替采样，计算差值得到瞬时包络值 ![](https://cdn.nlark.com/yuque/__latex/8eb33b4b02af08139ea61f3ca2696a4b.svg) 和 ![](https://cdn.nlark.com/yuque/__latex/619d7d4f4e7f3179c986c0b66c5d1fef.svg)。
5. 归一化与校准：去除直流偏置，归一化幅值（通常除以 ![](https://cdn.nlark.com/yuque/__latex/321138a59e6eab0c97c21f05282a80a6.svg)）得到标准的 ![](https://cdn.nlark.com/yuque/__latex/605d0e1cc65b9501482b9a107da21e4c.svg) 和 ![](https://cdn.nlark.com/yuque/__latex/6f68004a3cffdcd2dfadef558dc37e93.svg)。
6. 角度计算：调用 atan2(sinθ, cosθ) 或使用跟踪观测器（PLL） 计算出角度和速度。

总结：外包络的提取本质上是一个同步解调过程。在工程实践中，峰值采样法是实现软解码最轻量、最高效的方式，其核心难点在于ADC采样时序与硬件信号链延迟的精确匹配。如果硬件延迟补偿做不好，提取出的包络会出现幅值衰减和相位偏移，直接影响最终角度精度。
