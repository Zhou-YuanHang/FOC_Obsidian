## 2. 数字实现架构

在微控制器中实现相干解调，通常采用以下步骤：

### 2.1 载波同步

**参考信号的相位必须与接收到的载波相位严格一致**，否则会引入 ![](https://cdn.nlark.com/yuque/__latex/70c4e9dfd9b4ef085e031418b9d4ff24.svg) 衰减。对于旋变系统，励磁信号由MCU自身产生，因此可以利用产生励磁的定时器/计数器直接生成数字参考信号，天然实现同步。若硬件链路上有固定延迟，可在软件中补偿相位偏移。

### 2.2 数字乘法

ADC以固定采样率（通常远高于载波频率，如载波10kHz，采样率100kHz以上）采集 ![](https://cdn.nlark.com/yuque/__latex/983d1fd262527113202327da66eb4e50.svg) 和 ![](https://cdn.nlark.com/yuque/__latex/7cd43d57107c72c5c7ad362585b7da4e.svg)。在每个采样点，用预先计算或实时生成的参考正弦值 ![](https://cdn.nlark.com/yuque/__latex/4ad505d208c3863c5a190b3fe19b382c.svg) 与ADC值相乘。

### 2.3 低通滤波

由于乘法产生了二倍频分量，必须用数字低通滤波器将其滤除。常用滤波器：

- **IIR滤波器**（如二阶巴特沃斯）：计算量小，但存在相位非线性；
- **FIR滤波器**（如移动平均或等波纹设计）：线性相位，但阶数较高。

滤波器的截止频率需要设置在载波频率以下（通常低于载波频率的1/2），同时要保留包络的最高频率（即转子转速对应的最大频率）。

### 2.4 抽取与归一化

滤波后的数据速率可以降低（降采样），然后进行偏移校准、幅值归一化，得到 ![](https://cdn.nlark.com/yuque/__latex/605d0e1cc65b9501482b9a107da21e4c.svg) 和 ![](https://cdn.nlark.com/yuque/__latex/6f68004a3cffdcd2dfadef558dc37e93.svg)，最后通过反正切或观测器求角度。

## 3. 关键要点与注意事项

### 3.1 载波相位同步的重要性

若参考信号与载波存在相位差 ![](https://cdn.nlark.com/yuque/__latex/881403b34487b79bc2ceeeed2c8ea87c.svg)，解调出的包络会乘以 ![](https://cdn.nlark.com/yuque/__latex/70c4e9dfd9b4ef085e031418b9d4ff24.svg)，导致幅值衰减。如果两路（正弦、余弦）的相位差不同，还会引起正交性误差。因此，在实际系统中需对硬件延迟进行标定，并在参考信号中加入相移补偿。

### 3.2 低通滤波器的设计

- **带宽**：应略高于转子最高转速对应的电频率（例如最高转速下包络变化频率）。对于电机控制，通常取转速频率的3~5倍。
- **截止频率**：远低于载波频率，以保证对二倍频有足够的抑制。
- **延迟**：滤波器会引入群延迟，在高速控制中需考虑其对角度滞后及环路稳定性的影响。

### 3.3 计算复杂度

乘法运算和滤波（尤其是高阶FIR）会占用较多MCU资源。但现代MCU（如Cortex-M4/M7、C2000）大多带有硬件乘法器和DSP指令，可以高效执行。若使用IIR滤波器，计算量更小。

## 4. 与其他解调方法的比较

|  |  |  |  |
| --- | --- | --- | --- |
| 方法 | 核心操作 | 优点 | 缺点 |
| **相干解调** | 乘法 + 低通滤波 | 抗噪性能强；对采样点位置不敏感；可处理任意相位延迟 | 计算量大；需要滤波器设计；存在滤波延迟 |
| **峰值采样** | 在载波峰值处采样 | 计算量极小；实时性最高；无滤波器延迟 | 对采样时刻敏感；抗噪声能力弱；相位延迟补偿要求高 |
| **同步整流积分** | 半周期内累加（整流+积分） | 抗噪性优于峰值采样；对采样相位不敏感；计算量适中 | 需要较高采样率；需处理直流偏置；载波极性同步 |

## 5. 适用场景与工程建议

- **高精度要求**：当电机运行在低速或零速时，信噪比极低，峰值采样法易出现角度抖动，相干解调因滤波作用能提供更平滑的输出，适合伺服控制等高精度场合。
- **高带宽控制**：若转速极高，需考虑滤波器延迟。此时可选用低阶IIR滤波器，或改用同步整流积分法（延迟小）。
- **资源受限MCU**：若MCU算力紧张，优先选择峰值采样法或同步整流积分法；若MCU性能足够，相干解调可简化硬件设计（无需精确峰值对齐），提高系统鲁棒性。

---

## 6. 总结

相干解调通过乘法与低通滤波从调幅信号中恢复基带包络，是旋变软解码中理论基础最严密的方法。它不依赖于采样点的精确时序，而是通过频域处理提取有用信号，具有较好的抗噪声能力和对硬件延迟的容忍度。虽然计算量较大，但借助现代MCU的DSP能力，它仍然是实现高性能旋变软解码的可靠选择。

如果硬件延迟稳定且MCU资源紧张，峰值采样法或同步整流积分法更为高效；若追求鲁棒性、简化同步要求或系统噪声环境恶劣，相干解调是更优的方案。实际工程中，也可以结合多种方法的优势，例如用相干解调做低速下的角度观测，用峰值采样做高速下的快速响应。
